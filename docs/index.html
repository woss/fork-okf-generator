How It Works

Three steps to a smarter agent

From source code to structured knowledge graph in minutes. No LLM required.

1

Scan your codebase

tree-sitter AST parsers extract every function, class, module, and dependency across 17 languages with full signatures and docstrings.

okf generate ./my_project ./okf_bundle
2

Retrieve any concept

Exact-symbol lookup returns full concept cards — signature, parameters, callers, callees — in milliseconds. Zero embeddings, zero RAG.

okf lookup WorldBankConnector
3

Integrate with any agent

One command installs okf-generator into Claude Code, Cursor, Copilot, Windsurf, Cline, or OpenCode with auto-trigger rules.

okf install claude # or cursor, copilot…

LLM Enrichment

Go deeper when you need it

Four tiered enrichment modes. All are resumable — interrupt and rerun freely. Already-enriched concepts are skipped.

base

Base Enrichment

Improves descriptions and docstrings with Google-style formatting. Does not require source body — works on existing bundles.

okf generate ./src ./bundle --enrich base
deep

Deep Enrichment

Adds usage examples, side effects, security notes, and complexity estimates. Requires source body for full context.

okf enrich ./bundle --mode deep
security

Security Audit

Audits the bundle for visible risk patterns — injection vectors, auth bypasses, unsafe deserialization. Flags risks with remediation hints.

okf enrich ./bundle --mode security
full

Full Enrichment

All tiers plus semantic related-links across concepts. Best for training data generation and comprehensive architectural review.

okf enrich ./bundle --mode full

FAQ

Frequently asked questions

Does this require an API key or internet connection?
No. Core extraction (okf generate) is fully offline and deterministic — no LLM call is made unless you explicitly enable enrichment with --enrich. All 17 language parsers use tree-sitter or Python's stdlib AST and work completely air-gapped.
How is this different from RAG or vector search?
RAG retrieves chunks by semantic similarity, which is approximate and can miss exact symbols. okf lookup is exact: it indexes real functions, classes, modules, and dependencies by name and resolves to the precise concept with zero embedding or vector infrastructure required. The result is deterministic and identical every run.
Does this work on monorepos or very large codebases?
Yes. The bundle mirrors your source tree and scanning is linear in file count. For large repos, scope okf generate to a subdirectory if you only need part of the codebase indexed. Multi-bundle support lets you visualize cross-project edges with a bundle selector in the viz dashboard.
Is the bundle safe to commit to git?
Yes, and that is the intended workflow. Bundles are plain Markdown files with YAML frontmatter — they diff cleanly, version alongside the code they describe, and can be reviewed in any code review tool. The pre-commit hook auto-regenerates the bundle when source files change.
Can I use this without any LLM at all, ever?
okf generate + okf lookup together form a complete, zero-LLM workflow. LLM enrichment (okf enrich) and training pair synthesis (okf pairs --synth) are optional layers on top. You can get full value from okf-generator without ever providing an API key.
