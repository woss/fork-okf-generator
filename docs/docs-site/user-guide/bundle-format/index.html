    Bundle Format Reference¶

    An OKF bundle is a directory of plain Markdown files with YAML frontmatter, generated by okf generate. Every file describes one Concept — a class, function, module, dependency, or other unit extracted from your codebase. This page documents the full schema so you can parse, script against, or manually author bundle files.

    Directory Layout¶

    okf_bundle/
├── SUMMARY.md                  # Agent-facing index, regenerated by `okf summarize`
├── index.md                    # Bundle metadata (name, okf_version, timestamp)
├── log.md                      # Generation log
├── src/
│   └── mypackage/
│       ├── module.md            # One concept per file, mirrors source tree
│       └── utils/
│           └── helpers.md
└── _dependencies/
    └── pip/
        ├── requests.md
        └── fastapi.md
    • Source concepts mirror the path of the file they were extracted from, under the bundle root.
    • Dependency concepts live under _dependencies/{ecosystem}/{name}.md, one file per package.
    • SUMMARY.md is a generated, human/agent-readable table of contents — do not hand-edit it, run okf summarize to regenerate.

    Concept Frontmatter Schema¶

    Every concept file starts with YAML frontmatter followed by a Markdown body.

    ---
type: Function
title: slugify
resource: utils.py
tags:
  - lang:python
  - type:Function
  - module:utils
timestamp: 2026-07-01T10:22:03Z
signature: "def slugify(text: str, max_length: int = 80) -> str"
---
    Field Type Required Description
    type string yes Concept kind. See Concept Types below.
    title string yes Concept name (slugify, User, click)
    resource string yes Source file path relative to repo root
    tags list[string] yes Structured tags, see Tag Reference
    timestamp ISO 8601 yes Generation time
    signature string for functions/methods Full signature in language-native syntax
    description string no Short description (first line of docstring or enriched)

    Concept Types¶

    type Source
    Module A file or logical module
    Class A class, struct, or equivalent
    Function A top-level function or method
    Dependency An external package from a manifest file
    Interface Interfaces/protocols/traits
    Enum Enumeration types
    Constant Module-level constants
    Table SQL tables
    Trigger SQL triggers
    Type Type aliases

    Tag Reference¶

    Tags are flat strings in key:value form, used for filtering in okf lookup:

    Prefix Meaning Example
    lang: Source language lang:python, lang:rust
    type: Concept type (redundant with frontmatter, kept for grep) type:Function
    module: Owning module module:utils
    ecosystem: Dependency ecosystem ecosystem:pip
    version: Dependency version constraint version:==8.1.7
    manifest: Source manifest file manifest:requirements.txt
    git: Git metadata git:branch:main

    Dependency body_extra¶

    Dependency concepts include a markdown table with:

    Field Description
    Ecosystem Package ecosystem (pip, npm, cargo, etc.)
    Version constraint Version spec as declared
    Source manifest Filename the dependency was parsed from
    Dev dependency yes or no
    Used by Number of modules importing this dep

    Call Graph (markdown body)¶

    Functions and modules may include ## Calls / ## Called By / ## Related sections, generated by the linker:

    ## Calls
- `requests.get` (_dependencies/pip/requests)

## Called By
- `main.run_pipeline` (_dependencies/_modules/main)

## Related
- [slugify](utils/slugify.md)
- [chunk_list](utils/chunk_list.md)

    Minimal Valid Example¶

    ---
type: Function
title: add_numbers
resource: math_utils.py
tags:
  - lang:python
  - type:Function
  - module:math_utils
timestamp: 2026-07-01T09:00:00Z
signature: "def add_numbers(a: int, b: int) -> int"
---

Adds two integers and returns the sum.

    Versioning¶

    This schema targets OKF v0.1. Breaking changes will be called out in the Changelog.
